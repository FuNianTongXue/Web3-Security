# Frontend Design Review Checklist - Security Management Platform

## 1. 信息架构
- 是否按“发现风险 -> 研判 -> 处置 -> 复盘”组织导航。
- 是否支持从告警列表快速跳转到证据明细。
- 是否提供跨链/多项目视图切换。

## 2. 可读性与决策效率
- 是否区分风险等级颜色和形状，避免仅依赖颜色。
- 是否明确展示时间线、受影响资产、风险来源。
- 是否减少关键任务路径中的无效点击。

## 3. 安全操作防误触
- 是否对高风险操作提供二次确认。
- 是否记录操作前后状态快照。
- 是否在敏感操作中展示权限和影响范围。

## 4. 状态与反馈
- 是否覆盖空态、加载态、异常态。
- 是否为失败操作提供可执行下一步。
- 是否对批量操作提供进度与回执。

## 5. 性能与可扩展性
- 是否定义列表分页/虚拟滚动策略。
- 是否对大规模日志和事件流做增量渲染。
- 是否支持可配置的仪表板布局。

## 6. 可访问性与国际化
- 是否支持键盘导航和屏幕阅读器语义。
- 是否满足最小对比度要求。
- 是否支持时间、货币、地址格式本地化。

## 7. 设计交付完整性
- 是否包含页面流、组件规范、边界状态。
- 是否提供开发标注（尺寸、间距、状态图）。
- 是否附带验收清单和风险提示。
